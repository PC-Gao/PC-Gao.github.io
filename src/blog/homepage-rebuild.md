# 个人主页迁移到 mdBook

## 为什么迁移

原来的个人主页基于 Jekyll 学术模板，虽然能用，但目录结构偏重，包含论文、讲座、教学等大量不相关内容，不适合长期维护个人项目、博客和闲谈。

## 这次做了什么

- 改成真正的 mdBook 结构
- 用 `src/SUMMARY.md` 维护左侧目录树
- 把内容拆成项目、博客、闲谈三类章节
- 准备通过 GitHub Pages 自动构建和发布

## 现在怎么维护

- 项目内容写在 `src/projects/`
- 博客内容写在 `src/blog/`
- 闲谈内容写在 `src/notes/`

## 为什么 mdBook 更合适

对于个人长期维护的技术站点来说，mdBook 的目录组织、章节跳转和全文检索更接近“文档/手册”体验，比临时拼起来的首页模板更稳定。
